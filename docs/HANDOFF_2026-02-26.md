# Solar OpenEO Handoff (concise)

## Current Pipeline Status (2026-05-20)
- ONNX-based solar PV inference on CDSE is active.
- Single merged UDF performs SLIC temporal mosaic + ONNX inference per chunk (no intermediate cube shuffle).
- Process graph loads S2 L1C (13 bands) + L2A SCL, merges to a 14-band cube, and runs the merged UDF via `apply_neighborhood`.
- Output cube has 2 bands: `solar_pv` (uint8 binary), `solar_pv_probability` (float32). Band names are set by the UDF's `apply_metadata` and reasserted by a `rename_labels` step in the process graph.

## Active Defaults / Parameters
- Detection threshold: 0.80
- Neighborhood geometry: `size=192`, `overlap=32` (effective UDF input = 256×256, matches model input shape)
- Mosaic params: `snic_size_px=20`, `snic_compactness=1.0`, `clear_thresh=0.8`, `top_n_scenes=8`, `top_n_rescue=10`
- Job options: `executor-memory=6g`, `executor-memoryOverhead=3g`, `python-memory=disable`, `soft-errors=0.1`

## Normalization
- Mirrors `solar_ml.data.normalize_batch` with `mode='percentile'`:
  `x' = clip((x - p2) / max(p98 - p2, 1.0), 0, 1)`.
- Stats come from `band_stats.npz` mounted via the model dependency archive.
