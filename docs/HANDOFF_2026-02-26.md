# Solar OpenEO Handoff (concise)

## Current Pipeline Status (2026-05-12)
- ONNX-based solar PV inference on CDSE is active.
- Process graph uses S2 L1C + SCL-masked temporal median mosaic, then ONNX UDF via apply_neighborhood.
- UDF debug output currently includes four bands: solar_pv, solar_pv_probability, pre_norm_mean, post_norm_mean.
- UDF/model wiring issues are resolved (entrypoint detection + fixed-shape chunk geometry).
- 2026-05-13 fix: process graph now performs probability smoothing via positional band index from UDF output (index 1) to avoid Invalid band name/index errors on backends that keep source labels (B01..B12) after apply_neighborhood.

## Active Defaults / Parameters
- Threshold: 0.80
- Neighborhood geometry: size=192, overlap=32 (effective input = 256)
- Probability smoothing: disabled (2026-05-13 debug simplification)
- ONNX model/stats loaded from dependency archive mount under onnx_models/
- Drive exports remain explicit-folder based.

## Normalization + Output Behavior
- No fallback hardcoded stats in UDF.
- Inference normalization uses per-chip robust histogram alignment (p5�p95 trimmed mean/std per band).
- Optional override: UDF can consume `global_mean` + `global_std` (13 values each).
- Dynamic helper available for concrete AOI runs: `auto_global_stats=True` computes ROI-wide per-band mean/std from the same mosaic before inference and passes them to UDF context.
- Model output range verified in logs as probability in [0,1].
- Binary mask is rederived from smoothed probabilities in process graph.
